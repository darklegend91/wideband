# Folder Structure And Run Guide

This project contains old notebooks, exported scripts, trained model artifacts, predictions, and the final clean research workflow. For a new user, the important point is:

```text
Final code to run:
research_pipeline.py
compare_feedback_results.py
```

The `code/` and `workflow/` folders are useful references, but they are not the final beginner workflow.

For a detailed explanation of each machine learning technique, its normal use case, and its contribution to this antenna model, read:

```text
MODEL_TECHNIQUES_EXPLAINED.md
```

## Quick Start

From the project root:

```bash
source .venv/bin/activate
python research_pipeline.py
```

This trains the model using only the original dataset and writes results to:

```text
research_outputs/
```

To train with real simulation feedback:

```bash
python research_pipeline.py --feedback data/feedback_simulations.csv --output research_outputs_feedback
```

To compare original results vs feedback-trained results:

```bash
python compare_feedback_results.py
```

This writes comparison notes, tables, and plots to:

```text
comparison_outputs/
```

## `data/`

This folder contains input data.

```text
data/
├── UBW-data-final.xlsx
└── feedback_simulations.csv
```

### `UBW-data-final.xlsx`

This is the original simulation dataset. It must contain:

```text
SrNo, LG, WR, LR, WF, FL, FU, BW
```

The model learns from:

```text
Inputs:  LG, WR, LR, WF
Outputs: FL, FU, BW
Label:   resonant if BW > 0, dead if BW = 0
```

### `feedback_simulations.csv`

This is where you add real simulation results for model-recommended points.

The feedback file may include prediction columns, but the actual training values come from:

```text
Actual FL
Actual FU
Actual BW
```

A feedback row can look like:

```csv
LG,WR,LR,WF,physics_fc,Pred Freq(Lower) GHz,Pred Freq(Upper) GHz,Pred BW GHz,Probability resonant,Expected BW GHz,Combined score,Known in old dataset,Actual FL,Actual FU,Actual BW
5.5,12.25,3,3.6,7,5.273089641,8.307270882,3.236517259,0.567895806,1.838004578,1.549065158,FALSE,5.29,6.7,1.41
```

The script converts:

```text
Actual FL -> FL
Actual FU -> FU
Actual BW -> BW
```

Then it retrains the model using original data plus feedback data.

## Full Workflow For A New User

### 1. Setup Environment

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Run Baseline Model

```bash
python research_pipeline.py --output research_outputs
```

This creates baseline results using only:

```text
data/UBW-data-final.xlsx
```

### 3. Check Top Recommended Points

Open:

```text
research_outputs/tables/physics_guided_top_25_unseen_predictions.csv
```

Choose some recommended designs and simulate them in HFSS/CST.

### 4. Add Real Simulation Feedback

Add your simulation results to:

```text
data/feedback_simulations.csv
```

Make sure every row has:

```text
LG, WR, LR, WF, Actual FL, Actual FU, Actual BW
```

### 5. Retrain With Feedback

```bash
python research_pipeline.py --feedback data/feedback_simulations.csv --output research_outputs_feedback
```

This creates a new model that has learned from your real simulation feedback.

### 6. Compare Before And After

```bash
python compare_feedback_results.py
```

Open:

```text
comparison_outputs/feedback_comparison_report.md
```

Use the plots in:

```text
comparison_outputs/plots/
```

## What The Model Actually Does

The model does not directly calculate final bandwidth using only formulas.

It does this:

```text
1. Learns from existing simulated data.
2. Predicts whether a new antenna geometry is resonant.
3. Predicts FL, FU, and BW for candidate geometries.
4. Uses physics-guided wavelength-ratio scoring to rank physically plausible high-bandwidth candidates.
5. Accepts real simulator feedback and retrains to improve future predictions.
```

So the project is best described as:

```text
Physics-guided machine learning with simulation feedback.
```
